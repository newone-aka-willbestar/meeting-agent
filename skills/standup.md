# 站会抽取规则

你正在处理一场**每日站会（daily standup）**。站会的典型结构是每个人轮流讲三件事：
昨天做了什么、今天打算做什么、遇到什么阻塞。请据此抽取。

## 重点抽取（必须）
- **待办（action item）**：每个人"今天/接下来要做什么"。
  - 字段：`负责人`（必须能对应到具体的人）、`内容`、`ddl`（站会通常不说明确截止时间，缺失时填 null，别编造）。
- **风险 / 阻塞（blocker）**：谁被什么卡住了。站会里的"blocker"就归到风险。
  - 字段：`描述`（被什么卡住）、`相关负责人`、`影响`（可选）。

## 通常不抽（避免硬凑）
- **决策**：站会一般不做正式决策，没有就留空，不要为了填满而编。
- 与个人进度无关的闲聊、寒暄。

## 质量 rubric（自检 / 也是 eval 评判标准）
1. 每条待办都挂到了具体的人，没有"大家""团队"这种模糊负责人。
2. 阻塞写清了"被什么卡住"，而不是只写"有问题"。
3. 没有把"昨天已完成的事"误抽成待办。
4. 宁可漏抽不可错抽：不确定是不是 action item 的，不抽。

## 输出格式
按系统给定的 JSON schema 输出 `决策 / 待办 / 风险 / 待议` 四类；本类会议以 `待办` 和 `风险` 为主。
